# Instruction: Future Project Modifications

This file serves as the primary entry point and source of truth for AI agents tasked with implementing new features or modifications in this project. All AI agents must read this file first to understand context, status, and organization rules.

## 📁 Organization Rules

Each modification must follow this structure:

1.  **Folder Creation**: Create a folder under `modifications/` named after the feature (e.g., `modifications/FEATURE_NAME/`).
2.  **Context File**: Inside that folder, create a Markdown file (e.g., `FEATURE_NAME-Modification.md`) containing:
    - Business requirements.
    - Technical constraints.
    - Reference files/documentation.
3.  **Roadmap Update**: Update the "Status Roadmap" section in this file.

## 🤖 AI Onboarding Steps

When starting a task from this list:

1.  Locate the feature in the **Status Roadmap**.
2.  Follow the link to the internal documentation.
3.  Analyze any attached PDF or reference files in the feature folder.
4.  Map the requirements to the current codebase before proposing an implementation plan.
